## SAM3 `--no-deps` Undeclared Dependencies (pattern)

SAM3 is installed with `--no-deps` (error #5) to prevent it from downgrading numpy to <2.0 (which would break cuML). The cost: every runtime dep SAM3 uses must be declared explicitly in `requirements_gpu.txt`. Errors #6, #7, #18, #19 are all instances of this pattern.

| `einops` | Yes (`sam/rope.py`) | Fix #7 |
| `iopath` | Yes (`model_builder.py`) | Fix #18 |

| `regex` | Yes — but already transitive via transformers | Covered |

| `skimage`, `sklearn` | Yes (`visualization_utils.py`) — but already in base `requirements.txt` | Covered |

If another SAM3 dep error surfaces, same fix: add to `requirements_gpu.txt`, re-run `./setup_env_uv.sh --gpu --from-wheels`. Do NOT use `uv pip install` directly (CLAUDE.md rule: reproducibility).
